Com a guerra fria rolando, os kr da URSS criaram um satélite bem paia só para assustar os betas dos states, ai os states já tremeu e falo, fudeo, se os kr destruir essa base militar aq as informação tudo vai pra vala.
Ai que que os kr fez, vamo criar uma rede com os nossos computer e fazer com q eles passem dados entre si, pq ai se tudo explodir em uma os dados já tá no outro computador, essa rede era chamada de Arpanet

Arpanet

Só que os computador da epoca não coseguia se comunicar com um computador de outra marca, ai os kr tiveram que criar um protocólo pros computers poder trocar dados, o chamado NCP que fazia com que os computador pudessem se comunicar entre si e trocar informações

Beleza, só que era um protocolo bem but, se um computer tivesse que se comunicar com outro, a rede inteira parava pros dois se falar, e só depois que a comunicação acabasse, ai outro computer poderia se comunicar com outro

Ai dps que a rede cresceu mais, ngc fico impossivel de usar, imagina q é que nem sua internet bosta ai, ai quando tem mt kr usando ngc fica foda(apenas imagine, um ngc n tem nd a ver com outro, enquanto um tinha que parar pra poder comunicar com outro pc, sua internet é lenta por diversos fatores, como região, tem muitas antenas do seu plano ai? sua internet é de quanto Mb? etc...), ai o Bob Kahn cria o TCPque tinha como objetivo resolver esse problema de ter q parar a rede toda

Só q esse TCP ai tava com problema de identificação pq a rede cresceu dms, ai o Vint Cerf cria o IP, que é um identificador.
Ai os dois junto cria o TCP/IP que é usado até hj btw

Como os dados chegam no nosso computador?

  1. Computador A(cliente, nois) quer uma foto
  2. computador B(servidor) tem essa foto
  3. TCP/IP identifica onde tá o servidor q tem essa foto, com o IP
  4. O TCP/IP do servidor quebra a imagem pra ela ficar mais leve(principalmente quando é imagem grande), e manda pra nois por rota diferente, caminho, tempo tudo diferente, pq as vez algm caminho tá com problema, bla bla bla, ai o IP dele identifica o nosso computer pra poder enviar a imagem em pacotes no lugar certo
  5. O nosso TCP/IP monta a imagem, o TCP q faz isso e mostra pra nois na tela

O que é HTTP?

protocolo que permite mostra hypertextos(linguagem -> html) = imagens, textos, vídeos, gifs, etc; tudo junto

https

https é o http com criptografia, ou seja segurança dos seus dados

Representação de dados

Um computador funciona com sinais, sendo 0 e 1, ondas quadradas corretamente dito, ondas com sinais elétricos é o 1, sem é o 0
0 e 1 = binary digits, ou bit

  • 8 bits = 1 byte

Multiplos de bite

  1. 1024 bytes = 1 KB
  2. 1024 KB = 1 MB
  3. 1024 MB = 1 GB
  4. 1024 GB = 1 TB
  5. 1024 TB = 1 PB
  6. 1024 PB = 1 EB
  7. 1024 EP = 1 ZB
  8. 1024 ZB = 1 YB

Como nos conectamos?

  1. Cliente usa um serviço(telefone, internet, etc)
  2. Cliente digita "Instragram.com"
  3. Computador processa a informação, e envia onda quadradas pro modem
  4. O modem faz a modulação pra que a rede(que assinamos) consiga entender o que foi comunicado
  5. A rede procura o dominio no servidos DNS, onde funciona como um "contato de telefone"
  6. Após encontrar o "número" = IP do servidor
  7. O servidor irá mandar os dados do site de volta pra rede em pacotes(TCP), que manda para o modem
  8. Que faz a desmodulação para que o nosso computador consiga entender
  9. Nosso computador processa os dados e mostra na nossa tela, os dados q veio
